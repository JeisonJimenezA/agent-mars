llm-science-exam - LLM Science Exam Challenge

OBJECTIVE:
Given a science-based multiple-choice question generated by a Large Language Model (GPT-3.5),
predict the top 3 most probable correct answers from 5 options (A, B, C, D, E). This is a
multiple-choice question answering task where participants must rank their answer predictions
by confidence, and submissions are evaluated on whether the correct answer appears among
the top-3 predicted labels.

The broader goal is to advance understanding of LLM self-evaluation capabilities: can a
smaller model (≤10B parameters) correctly answer questions written by a much larger model
(GPT-3.5, 175B parameters)? This has direct implications for using LLMs as automated
benchmark generators and for deploying capable QA systems in resource-constrained environments.

CONTEXT AND MOTIVATION:
- State-of-the-art LLMs have rendered many existing NLP benchmarks trivially easy, motivating
  the use of LLMs themselves as question generators for harder, more discriminative benchmarks
- GPT-3.5 was given Wikipedia snippets on scientific topics and prompted to write multiple-choice
  questions with a known correct answer; easy questions were filtered out
- The Kaggle submission environment imposes GPU and time constraints, effectively capping
  usable model sizes at ~10B parameters — making this a genuine small-vs-large LLM challenge
- Questions span a broad range of scientific topics; the test set may exhibit distributional
  shift relative to the 200-question training sample, so generalization is critical
- This setup mirrors real-world resource-constrained inference scenarios (edge devices,
  quantized models, knowledge-distilled architectures)

DATASET:
Files:
- train.csv          : 200 sample questions with prompts, 5 answer options, and correct answer labels
- test.csv           : Test examples with prompts and options (no answers) — the re-run hidden
                       test set contains ~4,000 different prompts; predict ALL rows
- sample_submission.csv : Output format template only (DO NOT use as test data or to filter
                          test.csv — it only reflects a few sample IDs)

All files must be encoded as UTF-8

train.csv columns:
- prompt  : The text of the science question being asked (generated by GPT-3.5)
- A       : Text of answer option A
- B       : Text of answer option B
- C       : Text of answer option C
- D       : Text of answer option D
- E       : Text of answer option E
- answer  : The correct answer label as defined by the generating LLM (one of A, B, C, D, or E)

test.csv columns:
- id      : Unique identifier for each question
- prompt  : The text of the science question being asked
- A       : Text of answer option A
- B       : Text of answer option B
- C       : Text of answer option C
- D       : Text of answer option D
- E       : Text of answer option E

IMPORTANT DATA NOTES:
- The task is MULTIPLE-CHOICE: the model must select from exactly 5 provided options —
  no answer generation or span extraction is required
- The visible test.csv during development is a copy of training data without answers;
  the actual hidden re-run test set has ~4,000 different questions
- There may be distributional shift between the 200 training questions and the hidden
  test set — solutions that generalize across science topics are expected to perform better
- Questions were written natively by GPT-3.5 (not translated or adapted from existing benchmarks),
  making this a realistic adversarial evaluation rather than a recycled task
- The Kaggle submission environment enforces GPU and runtime limits — only models that
  can run within these constraints are viable; quantized or distilled models are encouraged

EVALUATION:
Metric: Mean Average Precision @ 3 (MAP@3)
Direction: Higher is better (maximize, range: 0.0 to 1.0)

MAP@3 is computed as:

  MAP@3 = (1/U) * Σ_{u=1}^{U} Σ_{k=1}^{min(n,3)} P(k) × rel(k)

Where:
- U      = number of questions in the test set
- P(k)   = precision at cutoff k (fraction of correct predictions in top-k)
- n      = number of predictions submitted per question (up to 3)
- rel(k) = indicator function: 1 if the item at rank k is the correct answer, 0 otherwise

Once a correct label has been scored for a given question, that label is no longer considered
relevant for additional predictions on that question (duplicate correct predictions are skipped).

Example: if the correct label is A, the following prediction lists all score AP = 1.0:
  [A, B, C, D, E]
  [A, A, A, A, A]
  [B, A, C, D, A]

SUBMISSION FORMAT:
For each id in the test set, predict up to 3 labels (space-separated) for the prediction column:

  id,prediction
  0,A B C
  1,B C A
  2,C A B
  etc.

CRITICAL NOTES ON THE METRIC:
- Only the top 3 predictions are evaluated — submitting more than 3 labels per row is invalid
- The order of predictions matters: the correct answer ranked 1st scores higher than ranked 3rd
- Submitting the correct answer at rank 1 always yields the maximum AP of 1.0 for that question
- Submitting no correct answer among the top 3 yields AP = 0.0 for that question
- Unlike accuracy, MAP@3 rewards confident correct predictions and partially rewards
  correct answers placed lower in the ranking
- Since there are 5 options and only 1 is correct, a random baseline achieves MAP@3 ≈ 0.488
  (expected value of ranking the correct answer among 3 of 5 options uniformly at random)
- Err on the side of placing your highest-confidence prediction first
