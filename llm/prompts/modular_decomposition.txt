==== Task ====
Design a modular Python repository to implement this ML solution.
You must decompose the solution into PYTHON FILES (modules).

# CRITICAL REQUIREMENTS

1. **Python File Names**: Each module name must be a valid Python filename
   (without .py extension). Use snake_case names like:
   - config (configuration and hyperparameters)
   - data_loader (data loading and preprocessing)
   - feature_engineering (feature creation and transformation)
   - model (model definition and training)
   - evaluator (evaluation metrics and validation)
   - utils (utility functions)
   - main (REQUIRED - entry point script)

2. **REQUIRED MODULE**: You MUST include a `main` module that:
   - Acts as the entry point (python main.py)
   - Orchestrates the full pipeline
   - Produces the final output/predictions

3. **Module Count**: Design 5-10 modules total (including main).
   - Prefer MORE smaller modules over FEWER large modules
   - Each module should have ONE clear responsibility
   - Split complex modules: e.g., instead of one huge "model.py", use
     "model_lgbm.py", "model_xgb.py", "model_ensemble.py"

4. **MODULE SIZE LIMIT (CRITICAL)**:
   - Each module description should result in code < 500 lines
   - If a module would be larger, SPLIT IT into multiple modules
   - Example splits:
     - feature_engineering → feature_basic.py + feature_advanced.py
     - model → model_base.py + model_lgbm.py + model_xgb.py
     - data_loader → data_loader.py + data_preprocessor.py

5. **Descriptions**: For each module, describe:
   - Purpose (what it does)
   - Key classes/functions to implement (2-4 max per module)
   - How it interacts with other modules
